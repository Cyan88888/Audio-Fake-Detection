`.npy`，训练阶段直接加载特征以加速实验迭代；<br>·**模型层**：`FrameTransformerDetector`（`safeear/models/detector_transformer.py`）将帧特征投影到嵌入维度后，送入 `TransformerClassifier`（`safeear/models/safeear.py`），包含多层 为核心检测指标（数据来源于各实验 `test_results.json` 中测试集字段；min-tDCF 计算依赖 ASVspoof 2019 LA 配套 ASV 分数，见 `safeear/utils/tdcf_metrics.py`）。其余指标（如 minDCF、Accuracy、AUC 等）可在终稿扩展表中补充。<br><br>---<br><br>### 二、已完成工作进展（截至阶段 S3）<br><br>**（1）数据集构建**<br><br>**（i）数据来源：**<br>·**公开基准数据集**：ASVspoof 2019 LA（Logical Access，逻辑访问场景）。<br>·**数据形态**：<br>① 原始波形为 **16 kHz、FLAC** 格式，按官方协议划分为 **训练集 / 开发集（验证）/ 评估集（测试）**；<br>② 协议文件提供 **Bonafide（真实）/ Spoof（伪造）** 二元标签，与说话人、攻击类型等信息对应（具体字段以官方 `cm` 协议为准）。<br>·**工程组织方式**：<br>① 使用 `datas/ASVSpoof2019/*.tsv` 列出各划分样本相对路径；<br>② 通过 `safeear/datas/asvspoof19.py` 中 `ASVSppof2019` 数据集类，将 **TSV + 协议 + 特征根目录** 关联为可训练样本；<br>③ 标签映射由协议文件解析得到（文件名/ utterance id → bonafide 或 spoof）。<br><br>**（ii）预处理与特征流程：**<br>·**波形与长度**：对过长语音按训练/验证/测试策略进行 **固定长度裁剪或填充**（配置中 `max_len=64600` 采样点，约 4 s）；与帧特征长度对齐（约 `max_len/320` 帧）。<br>·**离线特征**：采用 **WavLM-base** 提取帧级特征，脚本为 `datas/dump_wavlm_feature.py`，输出与每条 `.flac` 同路径的 **`.npy`**（形状约为 `[T, 768]`，加载后转为 `[C, T]` 供模型使用），实现 **“一次提特征、多次训模型”**，降低重复开销。<br>·**数据划分**：严格遵循 ASVspoof 2019 LA 官方 **train / dev / eval** 划分；**中期汇报的量化结果以官方 eval 测试集为准**。<br><br>**（2）模型开发**<br><br>**模型配置：**<br>·**前端（特征）**：固定维度 **768**（WavLM-base 隐藏维度）。<br>·**检测网络**：`FrameTransformerDetector` → `TransformerClassifier`（代码路径 `safeear/models/detector_transformer.py`、`safeear/models/safeear.py`）。<br>·**关键结构超参（消融范围）**：编码器层数、注意力头数、MLP 比例、Dropout/注意力 Dropout/Stochastic Depth、位置编码类型（正弦 / 可学习）、序列长度与 **注意力池化** 等，由 `config/ablation_s*.yaml` 分阶段配置。<br>·**训练系统**：`TransformerSpoofTrainer`（`safeear/trainer/transformer_trainer.py`），优化器一般为 **AdamW**，学习率约 **3e-4**，并配合 **早停（EarlyStopping）**、**模型检查点（ModelCheckpoint）** 按验证集 **val_eer** 保存最优权重。<br><br>**训练过程（代表性设置，以主实验配置为例）：**<br>·**数据加载**：`DataModule` batch_size 多为 **64**，`num_workers` 为 **8**（具体以各 `config.yaml` 为准）。<br>·**训练轮次**：单阶段实验 max_epochs 多为 **30**，实际有效轮次由早停决定。<br>·**验证与测试**：训练结束后使用 `test.py` 在 **同一配置** 下先 `validate` 再 `test`，结果写入 `Exps/<实验名>/test_results.json`。<br>·**模型保存策略**：以验证集
